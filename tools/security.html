Security & vulnerability scoring

Aggregated security intelligence — IP reputation, composite website grade, and CVE search. Run server-side because the API keys for the upstream sources (Shodan, AbuseIPDB, etc.) belong on the backend, not in browser JS.

IP Reputation & CVE Recon

Aggregated lookup across AbuseIPDB, GreyNoise, VirusTotal, Shodan. Returns a 0–100 risk score plus exposed-service CVEs.

Website Security Score

Combined grade A+ to F. Mozilla Observatory + SSL Labs + our own header / cookie / mixed-content checks. Fancy gauge included.

CVE / NVD Search

Search the NIST National Vulnerability Database by product, version, or CVE ID. CVSS scores, exploit links, affected versions.

IP Reputation

Passive lookup across multiple sources — no scanning of the target. GreyNoise Community always works (no key). AbuseIPDB activates when ABUSEIPDB_API_KEY is set on the backend (free tier: 1,000/day). Returns a composite 0–100 risk score with a per-source breakdown.

Website Security Score (Composite)

Two-source composite grade — runs our own header check (fast) and pulls Mozilla Observatory's cached scan in parallel (no fresh scan kicked off, ~2-3s total). Returns A+ to F with a fancy gauge plus the per-source breakdown.

Source: Mozilla HTTP Observatory (free, no API key) + ionet's own header grader. Future: chain SSL Labs grade.

CVE / NVD Search

Direct search of the NIST National Vulnerability Database. Search by keyword (e.g. nginx 1.18), product, or CVE ID (CVE-2024-3094). Results show CVSS score + severity, description, and a link to the full NVD record.
